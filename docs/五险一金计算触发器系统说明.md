# 五险一金计算触发器系统说明

## 📖 文档概述

本文档详细说明高新区财政局工资信息管理系统v3版本中的社会保险（五险一金）自动计算触发器系统的设计、实现和使用方法。

**适用版本**: Supabase v3系统  
**最后更新**: 2025-07-26  
**文档版本**: 2.0

## 🏗️ 系统架构

### 核心概念

五险一金计算系统基于 **触发器驱动的自动计算模式**，当工资记录插入或更新时，系统会自动：

1. 🔍 **识别员工身份类别**
2. 📋 **匹配适用的保险规则**  
3. 🧮 **执行保险费用计算**
4. 💾 **存储计算结果**
5. 📝 **记录计算日志**

### 系统组件

```
📊 工资数据录入
    ↓
🔧 payroll_items 触发器 (calculate_social_insurance_on_payroll)
    ↓
📋 保险规则匹配 (insurance_type_category_rules)
    ↓
🧮 分险种计算函数
    ├── 养老保险 (calculate_pension_insurance)
    ├── 医疗保险 (calculate_medical_insurance)  
    ├── 失业保险 (calculate_unemployment_insurance)
    ├── 工伤保险 (calculate_injury_insurance)
    ├── 生育保险 (calculate_maternity_insurance)
    ├── 职业年金 (calculate_occupational_pension)
    ├── 大病医疗 (calculate_serious_illness_insurance)
    └── 住房公积金 (calculate_housing_fund)
    ↓
💾 计算结果存储 (payroll_items)
    ↓
📝 计算日志记录 (insurance_calculation_logs)
```

## 📊 数据结构设计

### 核心数据表

#### 1. insurance_types - 保险类型定义
```sql
-- 定义所有保险类型
CREATE TABLE insurance_types (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,           -- 保险名称（如：养老保险）
    system_key VARCHAR(50) UNIQUE NOT NULL, -- 系统标识（如：pension）
    description TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**标准保险类型**：
- `pension` - 养老保险
- `medical` - 医疗保险  
- `unemployment` - 失业保险
- `injury` - 工伤保险
- `maternity` - 生育保险
- `occupational_pension` - 职业年金
- `serious_illness` - 大病医疗
- `housing_fund` - 住房公积金

#### 2. insurance_type_category_rules - 保险适用规则
```sql
-- 定义各保险类型对不同员工类别的适用规则和费率
CREATE TABLE insurance_type_category_rules (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    insurance_type_id UUID REFERENCES insurance_types(id),
    employee_category_id UUID REFERENCES employee_categories(id),
    employee_rate NUMERIC(5,4) NOT NULL,     -- 个人费率
    employer_rate NUMERIC(5,4) NOT NULL,     -- 单位费率
    is_applicable BOOLEAN DEFAULT TRUE,      -- 是否适用
    effective_date DATE DEFAULT CURRENT_DATE,
    end_date DATE,
    description TEXT
);
```

#### 3. payroll_items - 工资明细（含保险计算结果）
```sql
-- 工资明细表，包含五险一金计算结果
CREATE TABLE payroll_items (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    payroll_id UUID REFERENCES payrolls(id),
    employee_id UUID REFERENCES employees(id),
    component_id UUID REFERENCES salary_components(id),
    amount NUMERIC(10,2) NOT NULL,           -- 金额
    calculation_base NUMERIC(10,2),          -- 计算基数
    rate NUMERIC(5,4),                       -- 费率
    is_employee_contribution BOOLEAN DEFAULT FALSE, -- 是否为个人缴费
    calculation_method TEXT,                 -- 计算方法
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 4. employee_contribution_bases - 员工缴费基数
```sql
-- 员工特定保险的缴费基数
CREATE TABLE employee_contribution_bases (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    employee_id UUID REFERENCES employees(id),
    insurance_type_id UUID REFERENCES insurance_types(id),
    contribution_base NUMERIC(10,2) NOT NULL, -- 缴费基数
    effective_start_date DATE NOT NULL,
    effective_end_date DATE,
    UNIQUE(employee_id, insurance_type_id, effective_start_date)
);
```

## ⚙️ 触发器系统实现

### 主触发器 - calculate_social_insurance_on_payroll

**触发条件**: 当 `payroll_items` 表插入或更新基础工资记录时

```sql
CREATE OR REPLACE FUNCTION calculate_social_insurance_on_payroll()
RETURNS TRIGGER AS $$
DECLARE
    v_employee_id UUID;
    v_payroll_id UUID;
    v_gross_salary NUMERIC;
    v_employee_category_id UUID;
BEGIN
    -- 仅处理基本工资组件
    IF NEW.component_id = (SELECT id FROM salary_components WHERE system_key = 'gross_salary') THEN
        v_employee_id := NEW.employee_id;
        v_payroll_id := NEW.payroll_id;
        v_gross_salary := NEW.amount;
        
        -- 获取员工类别
        SELECT ec.id INTO v_employee_category_id
        FROM employee_category_assignments eca
        JOIN employee_categories ec ON eca.category_id = ec.id
        WHERE eca.employee_id = v_employee_id
        AND CURRENT_DATE BETWEEN eca.effective_start_date 
        AND COALESCE(eca.effective_end_date, '9999-12-31'::DATE)
        LIMIT 1;
        
        -- 执行各险种计算
        PERFORM calculate_pension_insurance(v_employee_id, v_payroll_id, v_gross_salary, CURRENT_DATE);
        PERFORM calculate_medical_insurance(v_employee_id, v_payroll_id, v_gross_salary, CURRENT_DATE);
        PERFORM calculate_unemployment_insurance(v_employee_id, v_payroll_id, v_gross_salary, CURRENT_DATE);
        PERFORM calculate_injury_insurance(v_employee_id, v_payroll_id, v_gross_salary, CURRENT_DATE);
        PERFORM calculate_maternity_insurance(v_employee_id, v_payroll_id, v_gross_salary, CURRENT_DATE);
        PERFORM calculate_occupational_pension(v_employee_id, v_payroll_id, v_gross_salary, CURRENT_DATE);
        PERFORM calculate_serious_illness_insurance(v_employee_id, v_payroll_id, v_gross_salary, CURRENT_DATE);
        PERFORM calculate_housing_fund(v_employee_id, v_payroll_id, v_gross_salary, CURRENT_DATE);
    END IF;
    
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- 创建触发器
CREATE TRIGGER trigger_calculate_social_insurance
    AFTER INSERT OR UPDATE ON payroll_items
    FOR EACH ROW
    EXECUTE FUNCTION calculate_social_insurance_on_payroll();
```

## 🧮 保险计算函数详解

### 通用计算模式

每个保险计算函数都遵循统一的计算模式：

```sql
CREATE OR REPLACE FUNCTION calculate_[insurance_type](
    p_employee_id UUID,
    p_payroll_id UUID, 
    p_contribution_base NUMERIC,
    p_effective_date DATE
) RETURNS VOID AS $$
DECLARE
    v_employee_category_id UUID;
    v_employee_rate NUMERIC(5,4);
    v_employer_rate NUMERIC(5,4);
    v_is_applicable BOOLEAN;
    v_employee_amount NUMERIC(10,2);
    v_employer_amount NUMERIC(10,2);
    v_component_id UUID;
BEGIN
    -- 1. 获取员工类别
    SELECT ec.id INTO v_employee_category_id
    FROM employee_category_assignments eca
    JOIN employee_categories ec ON eca.category_id = ec.id
    WHERE eca.employee_id = p_employee_id
    AND p_effective_date BETWEEN eca.effective_start_date 
    AND COALESCE(eca.effective_end_date, '9999-12-31'::DATE)
    LIMIT 1;
    
    -- 2. 查询保险规则
    SELECT employee_rate, employer_rate, is_applicable
    INTO v_employee_rate, v_employer_rate, v_is_applicable
    FROM insurance_type_category_rules itcr
    JOIN insurance_types it ON itcr.insurance_type_id = it.id
    WHERE it.system_key = '[insurance_key]'
    AND itcr.employee_category_id = v_employee_category_id
    AND itcr.is_applicable = TRUE
    AND p_effective_date BETWEEN itcr.effective_date 
    AND COALESCE(itcr.end_date, '9999-12-31'::DATE)
    LIMIT 1;
    
    -- 3. 跳过不适用的保险
    IF NOT v_is_applicable OR v_employee_rate IS NULL THEN
        RETURN;
    END IF;
    
    -- 4. 计算保险费用
    v_employee_amount := p_contribution_base * v_employee_rate;
    v_employer_amount := p_contribution_base * v_employer_rate;
    
    -- 5. 特殊处理（如住房公积金进位规则）
    -- [特定保险的特殊逻辑]
    
    -- 6. 存储计算结果
    -- [插入 payroll_items 记录]
    
    -- 7. 记录计算日志
    -- [插入 insurance_calculation_logs 记录]
END;
$$ LANGUAGE plpgsql;
```

### 特殊计算规则

#### 住房公积金进位规则

```sql
-- 住房公积金特殊进位函数
CREATE OR REPLACE FUNCTION apply_housing_fund_rounding(amount NUMERIC)
RETURNS NUMERIC AS $$
BEGIN
    RETURN CASE 
        WHEN (amount - FLOOR(amount)) >= 0.10 THEN CEIL(amount)
        ELSE FLOOR(amount)
    END;
END;
$$ LANGUAGE plpgsql;
```

**规则说明**：
- 小数部分 ≥ 0.10：向上取整
- 小数部分 < 0.10：向下取整

#### 失业保险特殊规则

对于以下人员类别**不缴纳失业保险**：
- 公务员 (75)
- 机关工勤 (81) 
- 参照公务员管理 (71)
- 事业编制 (91)

#### 职业年金适用范围

**仅适用于正式编制人员**：
- 事业工勤人员 (82)
- 事业管理人员 (70)
- 事业技术工人 (90)
- 公务员 (75)
- 机关工勤 (81)
- 参照公务员管理 (71)
- 事业编制 (91)

**不适用于聘用人员**：
- 执业类专技人员 (78)
- 管理类专技人员 (83)
- 项目服务专员 (86)
- 等其他聘用类别

## 🔧 系统配置和维护

### 保险费率配置

#### 查看当前费率配置
```sql
SELECT 
    it.name AS 保险类型,
    ec.name AS 人员类别,
    itcr.employee_rate AS 个人费率,
    itcr.employer_rate AS 单位费率,
    itcr.is_applicable AS 是否适用
FROM insurance_type_category_rules itcr
JOIN insurance_types it ON itcr.insurance_type_id = it.id
JOIN employee_categories ec ON itcr.employee_category_id = ec.id
WHERE itcr.is_applicable = TRUE
ORDER BY it.name, ec.name;
```

#### 更新费率配置
```sql
-- 更新住房公积金费率为12%
UPDATE insurance_type_category_rules 
SET employee_rate = 0.1200,
    employer_rate = 0.1200
WHERE insurance_type_id = (
    SELECT id FROM insurance_types WHERE system_key = 'housing_fund'
);
```

### 缴费基数管理

#### 批量设置员工缴费基数
```sql
-- 为所有员工设置2025年1月住房公积金缴费基数
INSERT INTO employee_contribution_bases (
    employee_id, 
    insurance_type_id, 
    contribution_base, 
    effective_start_date
)
SELECT 
    e.id,
    (SELECT id FROM insurance_types WHERE system_key = 'housing_fund'),
    21200.00, -- 统一基数
    '2025-01-01'
FROM employees e
WHERE e.is_active = TRUE
ON CONFLICT (employee_id, insurance_type_id, effective_start_date) 
DO UPDATE SET contribution_base = EXCLUDED.contribution_base;
```

## 🚨 故障排除

### 常见问题

#### 1. 保险费用未自动计算

**可能原因**：
- 触发器未正确触发
- 员工类别配置缺失
- 保险规则配置错误

**排查步骤**：
```sql
-- 1. 检查触发器是否存在
SELECT * FROM pg_trigger WHERE tgname = 'trigger_calculate_social_insurance';

-- 2. 检查员工类别分配
SELECT * FROM employee_category_assignments 
WHERE employee_id = '<employee_id>'
AND CURRENT_DATE BETWEEN effective_start_date AND COALESCE(effective_end_date, '9999-12-31');

-- 3. 检查保险规则配置
SELECT * FROM insurance_type_category_rules itcr
JOIN insurance_types it ON itcr.insurance_type_id = it.id
WHERE itcr.employee_category_id = '<category_id>'
AND itcr.is_applicable = TRUE;
```

#### 2. 计算金额与预期不符

**排查步骤**：
```sql
-- 检查计算日志
SELECT * FROM insurance_calculation_logs
WHERE employee_id = '<employee_id>'
ORDER BY created_at DESC;

-- 检查缴费基数
SELECT * FROM employee_contribution_bases
WHERE employee_id = '<employee_id>'
AND CURRENT_DATE BETWEEN effective_start_date AND COALESCE(effective_end_date, '9999-12-31');
```

#### 3. 住房公积金进位错误

**验证进位函数**：
```sql
-- 测试进位函数
SELECT 
    1234.09 AS 原值,
    apply_housing_fund_rounding(1234.09) AS 进位后,
    1234.10 AS 原值2,
    apply_housing_fund_rounding(1234.10) AS 进位后2;
```

### 系统重置

如需重新计算所有保险费用：

```sql
-- 1. 删除现有保险计算结果
DELETE FROM payroll_items 
WHERE component_id IN (
    SELECT id FROM salary_components 
    WHERE system_key LIKE '%_employee' OR system_key LIKE '%_employer'
);

-- 2. 清空计算日志
DELETE FROM insurance_calculation_logs;

-- 3. 重新触发计算（更新基础工资记录）
UPDATE payroll_items 
SET amount = amount 
WHERE component_id = (SELECT id FROM salary_components WHERE system_key = 'gross_salary');
```

## 📈 系统监控

### 性能监控

#### 计算统计
```sql
-- 每日计算统计
SELECT 
    DATE(created_at) AS 日期,
    COUNT(*) AS 计算次数,
    COUNT(DISTINCT employee_id) AS 涉及员工数,
    COUNT(CASE WHEN error_message IS NOT NULL THEN 1 END) AS 错误次数
FROM insurance_calculation_logs
WHERE created_at >= CURRENT_DATE - INTERVAL '7 days'
GROUP BY DATE(created_at)
ORDER BY 日期 DESC;
```

#### 数据完整性检查
```sql
-- 检查是否有员工缺少保险计算结果
SELECT 
    e.name AS 员工姓名,
    COUNT(pi.id) AS 保险项目数
FROM employees e
LEFT JOIN payroll_items pi ON e.id = pi.employee_id
LEFT JOIN salary_components sc ON pi.component_id = sc.id
WHERE sc.system_key LIKE '%_employee'
AND pi.payroll_id = '<current_payroll_id>'
GROUP BY e.id, e.name
HAVING COUNT(pi.id) < 8  -- 应该有8个保险项目
ORDER BY COUNT(pi.id);
```

### 告警规则

建议设置以下监控告警：

1. **计算失败告警**: 当 `insurance_calculation_logs` 表中出现错误记录时
2. **数据完整性告警**: 当员工缺少应有的保险计算结果时
3. **费率变更告警**: 当 `insurance_type_category_rules` 表发生更新时

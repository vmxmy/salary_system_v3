# 薪资系统触发器和函数详细说明文档

## 概述

本文档详细说明了薪资管理系统中的所有触发器函数和RPC函数，包括369个RPC函数和30个触发器函数，涵盖了员工管理、薪资计算、社会保险、数据维护等核心业务逻辑。该文档基于实际数据库查询结果，为系统维护和开发提供完整的函数参考。

## 数据库触发器完整列表

### 1. 薪资计算触发器

#### 1.1 payroll_items表触发器组
**总收入计算触发器：**
- **gross_pay_trigger_on_insert** - INSERT后触发
- **gross_pay_trigger_on_update** - UPDATE后触发  
- **gross_pay_trigger_on_delete** - DELETE后触发
- **调用函数**: `calculate_gross_pay`
- **功能**: 薪资项目变更时自动重新计算员工总收入

**总扣除计算触发器：**
- **total_deductions_trigger_on_insert** - INSERT后触发
- **total_deductions_trigger_on_update** - UPDATE后触发
- **total_deductions_trigger_on_delete** - DELETE后触发
- **调用函数**: `calculate_total_deductions`
- **功能**: 扣除项目变更时自动重新计算员工总扣除

#### 1.2 payrolls表触发器
- **net_pay_trigger_on_update** - UPDATE时触发
- **调用函数**: `calculate_net_pay`
- **功能**: 根据总收入和总扣除计算净收入

- **payroll_paid_trigger** - UPDATE时触发
- **调用函数**: `notify_employee_on_payroll_paid`
- **功能**: 薪资状态变更为已发放时通知员工

### 2. 员工信息维护触发器

#### 2.1 employees表触发器组
**身份证号处理触发器：**
- **set_birthday_on_insert** - INSERT前触发
- **update_birthday_on_id_change** - UPDATE前触发
- **调用函数**: `update_birthday_from_id_card`
- **功能**: 根据身份证号自动提取并更新生日

- **set_gender_on_insert** - INSERT前触发
- **update_gender_on_id_change** - UPDATE前触发
- **调用函数**: `update_gender_from_id_card`
- **功能**: 根据身份证号自动提取并更新性别

### 3. 保险计算触发器

#### 3.1 employee_contribution_bases表触发器组
**保险重新计算触发器：**
- **recalculate_insurances_on_base_insert** - INSERT后触发
- **recalculate_insurances_on_base_update** - UPDATE后触发
- **recalculate_insurances_on_base_delete** - DELETE后触发
- **调用函数**: `trigger_recalculate_insurances_on_base_change`
- **功能**: 缴费基数变更时自动重新计算所有相关社会保险

- **trigger_insurance_calculation_on_contribution_base_change** - INSERT后触发
- **调用函数**: `trigger_calculate_and_update_insurance`
- **功能**: 缴费基数变更时触发保险计算更新

#### 3.2 insurance_calculation_logs表触发器
- **trigger_insurance_calculation_on_logs_change** - INSERT后触发
- **调用函数**: `trigger_calculate_and_update_insurance`
- **功能**: 保险计算日志变更时触发相关更新

### 4. 员工类别管理触发器

#### 4.1 employee_category_assignments表触发器组
- **recalculate_on_category_change** - INSERT后触发
- **调用函数**: `trigger_employee_category_change`
- **功能**: 员工类别变更时触发相关业务逻辑重新计算

- **trg_check_category_assignment_overlap** - INSERT前触发
- **调用函数**: `check_category_assignment_overlap`
- **功能**: 防止员工在同一时间段内被分配多个冲突类别

### 5. 系统维护触发器

#### 5.1 时间戳更新触发器
- **update_insurance_type_category_rules_updated_at** (insurance_type_category_rules表)
- **trigger_update_personal_income_tax_calculation_logs_updated_at** (personal_income_tax_calculation_logs表)
- **调用函数**: `update_updated_at_column`
- **功能**: 自动更新记录的最后修改时间

#### 5.2 政策规则变更触发器
- **log_policy_rule_changes** (policy_rules表) - INSERT后触发
- **调用函数**: `trigger_policy_rule_change`
- **功能**: 记录政策规则变更历史，用于审计追踪

## 主要RPC函数分类

### 1. 保险计算函数（Insurance Calculation）

#### 1.1 基础保险计算
- **calculate_employee_social_insurance**: 计算员工社会保险
- **calculate_employee_social_insurance_v2**: 社会保险计算V2版本
- **calculate_monthly_insurance_with_eligibility**: 带资格检查的月度保险计算

#### 1.2 具体保险类型计算
- **calculate_pension_insurance**: 养老保险计算
- **calculate_medical_insurance**: 医疗保险计算
- **calculate_unemployment_insurance**: 失业保险计算
- **calculate_work_injury_insurance**: 工伤保险计算
- **calculate_maternity_insurance**: 生育保险计算
- **calculate_housing_fund**: 住房公积金计算
- **calculate_serious_illness_insurance**: 大病保险计算
- **calculate_occupational_pension**: 职业年金计算

#### 1.3 批量保险计算
- **batch_calculate_social_insurance**: 批量计算社会保险
- **batch_recalculate_social_insurances**: 批量重新计算社会保险

#### 1.4 保险资格检查
- **check_insurance_eligibility**: 检查保险资格
- **check_insurance_applicability**: 检查保险适用性
- **auto_zero_ineligible_contribution_bases**: 自动将不合格基数设为0

### 2. 缴费基数管理函数（Contribution Base Management）

#### 2.1 基数调整
- **adjust_insurance_base**: 调整保险基数
- **apply_base_limits**: 应用基数上下限
- **apply_housing_fund_rounding**: 住房公积金舍入处理

#### 2.2 基数创建和维护
- **create_next_month_bases**: 创建下月基数
- **convert_open_to_closed_records**: 开放记录转封闭记录

### 3. 薪资管理函数（Payroll Management）

#### 3.1 薪资批量操作
- **create_payroll_batch**: 批量创建薪资记录
- **copy_payroll_data_from_source**: 从源期间复制薪资数据
- **clear_payroll_data_by_period**: 按期间清空薪资数据

### 4. 员工管理函数（Employee Management）

#### 4.1 员工导入
- **batch_import_employees_from_pgs**: 从PGS系统批量导入员工
- **batch_import_diverse_employees**: 批量导入多样化员工
- **batch_import_remaining_employees**: 批量导入剩余员工
- **batch_import_test_employees**: 批量导入测试员工

#### 4.2 联系信息和银行信息
- **batch_import_all_contacts_banks**: 批量导入联系方式和银行信息
- **batch_import_contacts_and_banks**: 批量导入联系方式和银行账户

#### 4.3 员工类别管理
- **assign_employee_category_batch**: 批量分配员工类别

### 5. 系统管理函数（System Management）

#### 5.1 权限管理
- **can_access_employee**: 检查员工访问权限  
- **can_view_sensitive_data**: 检查敏感数据查看权限
- **create_admin_user**: 创建管理员用户

#### 5.2 数据维护
- **cleanup_change_log**: 清理变更日志
- **cleanup_old_logs**: 清理旧日志
- **clean_duplicate_insurance_calculations**: 清理重复保险计算记录

#### 5.3 系统诊断
- **check_rls_coverage**: 检查RLS覆盖情况

### 6. 映射和缓存函数（Mapping and Cache）

#### 6.1 映射管理
- **auto_refresh_mappings_batch**: 自动刷新映射批处理
- **track_mapping_changes**: 跟踪映射变更

## 函数使用建议

### 1. 保险计算流程
```sql
-- 推荐的保险计算流程
1. check_insurance_eligibility() -- 检查资格
2. calculate_monthly_insurance_with_eligibility() -- 计算月度保险
3. batch_recalculate_social_insurances() -- 批量重算（如需要）
```

### 2. 薪资处理流程
```sql
-- 推荐的薪资处理流程
1. create_payroll_batch() -- 创建薪资批次
2. copy_payroll_data_from_source() -- 复制基础数据（可选）
3. 触发器自动计算总收入、总扣除和净收入
```

### 3. 数据维护流程
```sql
-- 推荐的数据维护流程
1. cleanup_old_logs() -- 定期清理日志
2. clean_duplicate_insurance_calculations() -- 清理重复计算
3. check_rls_coverage() -- 检查权限覆盖
```

## 注意事项

1. **触发器执行顺序**: BEFORE触发器在AFTER触发器之前执行
2. **性能考虑**: 批量操作时注意触发器可能带来的性能影响
3. **数据一致性**: 依赖触发器的自动计算确保数据一致性
4. **权限控制**: RPC函数通过RLS策略控制数据访问权限
5. **错误处理**: 函数中包含完善的错误处理和回滚机制

## 系统架构特点

1. **事件驱动**: 大量使用触发器实现事件驱动的数据处理
2. **自动化计算**: 薪资和保险计算高度自动化
3. **数据完整性**: 通过触发器确保关联数据的一致性
4. **审计追踪**: 完整的变更历史记录和审计日志
5. **批量处理**: 支持大规模数据的批量导入和处理
